# Система підтримки прийняття рішень — Аналіз інвестиційного ринку

Короткий опис
------------
Цей університетський проєкт реалізує Систему підтримки прийняття рішень (СППР) для напівструктурованої задачі — аналізу інвестиційного ринку та вибору найкращого активу для формування портфеля. Система надає REST API, яке обчислює рейтинг альтернатив за набором критеріїв та стратегією агрегування.

Технічний стек
--------------
- Мова: C#, .NET 9
- Web: ASP.NET Core Web API
- Документація API: Scalar
- Архітектура: N‑Tier (Controllers, Services, Data) + Dependency Injection

Структура проєкту (коротко)
---------------------------
- `Controllers` — HTTP-ендпоїнти (API). Приймають запити та повертають результати.
- `Services` — аналітичне ядро (DSS engine): нормалізація, обчислення згорток (SAW, MEW) та підготовка пояснень.
- `Data` — імітація сховища даних (`DssContext`): визначення альтернатив та критеріїв, початкові дані.

Формалізація задачі
-------------------
Множина альтернатив A (у проєкті захардкоджено 4):
- Технологічна компанія
- Стартап
- Державні облігації
- Дивідендний аристократ

Множина критеріїв C та їхні ваги:
1. Очікувана дохідність — напрям: Максимізація, вага = 0.40
2. Волатильність / Ризик — напрям: Мінімізація, вага = 0.30
3. Дивідендна дохідність — напрям: Максимізація, вага = 0.15
4. Коефіцієнт P/E — напрям: Мінімізація, вага = 0.15

Аналітичне ядро (нормалізація та методи згортки)
------------------------------------------------
1. Нормалізація
   - Перед агрегацією всі сирі числові оцінки приводяться до інтервалу [0, 1] по кожному критерію, щоб зробити їх сумісними.
   - Якщо для критерію c значення для альтернатив мають мінімум min та максимум max, то:
     - Для критерію з напрямом Maximize:

       norm(x) = (x - min) / (max - min)

     - Для критерію з напрямом Minimize:

       norm(x) = (max - x) / (max - min)

   - Якщо max == min (відсутність варіації), використовується політика уникнення ділення на нуль (наприклад, присвоєння 0.0).

2. SAW (Simple Additive Weighting) — проста адитивна згортка
   - Для кожної альтернативи a обчислюється зважена сума нормалізованих показників:

     Score_SAW(a) = Σ_{i ∈ C} w_i * norm_i(a)

   - Стратегія за замовчуванням: `SAW`.

3. MEW (Multiplicative Exponential Weighting) — мультиплікативна згортка
   - Для кожної альтернативи обчислюється добуток нормалізованих показників у степенях ваг:

     Score_MEW(a) = Π_{i ∈ C} (norm_i(a))^{w_i}

   - Щоб уникнути повного занулення через нульові нормалізовані значення, застосовується мала поправка (ε) або захист через `max(epsilon, norm)`.

Пояснення рішення
-----------------
API повертає не лише рейтинг (score), а й текстове пояснення — наприклад, який критерій найбільше сприяв перемозі альтернативи. Пояснення базується на найбільшому внеску (w_i * norm_i) для обраної (найкращої) альтернативи.

Запуск проєкту
--------------
1. Відкрийте термінал у корені репозиторію (де знаходиться `InvestmentDecisionDss.sln`).
2. Перейдіть у каталог проекту та запустіть:

```powershell
cd InvestmentDecisionDss
dotnet run
```

3. Документація Scalar (після запуску сервера):

```
http://localhost:5005/scalar/v1
```

Примітка: у залежності від локальних налаштувань Kestrel порти можуть відрізнятися; перевірте вивід `dotnet run` для точного URL.

API: приклад використання
-------------------------
- Ендпоїнт: `GET /api/decision/evaluate`
- Query-параметр: `strategy` — `"SAW"` (за замовчуванням) або `"MEW"`

Приклад виклику (PowerShell):
```powershell
Invoke-RestMethod -Uri "http://localhost:5000/api/decision/evaluate?strategy=SAW" -Method Get
```

Приклад відповіді (JSON):
```json
{
  "results": {
    "Стартап": 0.732,
    "Технологічна компанія": 0.541,
    "Дивідендний аристократ": 0.412,
    "Державні облігації": 0.201
  },
  "bestAlternative": "Стартап",
  "explanation": "Перемога альтернативи 'Стартап' пояснюється її високою оцінкою по критерію 'Очікувана дохідність' (внесок: 0.293)."
}
```
